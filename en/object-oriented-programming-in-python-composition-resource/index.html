Object-oriented programming in Python: How to do composition in Python

6 de May de 201829 de May de 2018 JoaoMarcolanPython

In the last post we discussed the initial aspects of object-oriented programming . And another post how to create a class and how to instantiate it to create an object. Classes in isolation would not provide much insight into how a system is designed [6]. In this tutorial we will dive into how to the composition class relations works in Python. And discuss how to use this relation off object-oriented programming .

Relationships between classes.

Isolated classes do not provide much functionality to a system. Generally the classes that make up a software have relationships with other classes. The great advantage of inter-class relationships is that you can create more complex classes using objects from less complex classes.

The Unified Modeling Language (UML) provides several ways to represent relationships between classes. Each relationship represents a different type of connection between classes. However, in this tutorial we will discuss only the relationship of composition between classes. [6]

When to Use the Composite Relationship in Object-Oriented Programming

Composition is usually a good choice when an object is part of another object. For example, a calculator has components that are part of it (display, keyboard, logical unit) that can be programmed differently. At the end of the programming of these less complex classes we can use them to create the calculator. The composition relationship usually found when a class A (less complex) “… is part of …” a class B (more complex).

Calculator Composition Discussion

In summary, the calculator class has been able to add and subtract as expected plus have control of your battery. In the calculator class we encapsulate the lower-level details in other classes (batter, keyboard, display) and use instances (objects) of these classes to construct a more complex component (calculator). In this way we have been able to create a component (calculator) of a higher level of complexity without having to worry about the details of lower level. The calculator class can be part of a program with more complexity (e.g., track controller of a missile).

Encapsulation, Inheritance in Python and Polymorphism

The composition feature allows us to create more complex components by creating the simplest components. To use this feature simply make simple queries such as: – “the class “A” has a object “B”,C and D ?” – ” what the class A has ? . Using this feature quickly can make it difficult to know which methods can be used by other classes and which methods are important for the inner workings of the class.

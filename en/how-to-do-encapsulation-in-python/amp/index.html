How to do encapsulation in python

An important part of object-oriented programming is the encapsulation feature. This feature involves grouping methods and attributes into packages that contain different forms of access. For example, certain methods and attributes are packaged as public and can be accessed by any class other packets will have some access restriction â€“ methods and attributes can only be accessed by a child class or no other class can access.

Such an approach allows the programmer to easily create an application interface for the classes and also so that they can hide the methods that are used for the operation itself. For example, the methods used for class operation are defined as private and the methods that are to be used by more complex classes are defined as public. We have to remember that the main effort of the object-oriented programming paradigm is to organize the code and the encapsulation feature is aligned in this effort. But in Python there are no access modifiers.

But Python has no access modifier

In many languages (e.g. c ++, java) there is the concept of access modifiers. For example, in C ++ the private modifier determines that methods and attributes can only be accessed by the class itself; the protected modifier determines that only child classes can access; the public modifier determines that any class can access the elements.

Private Variables in Python

Python does not have the concept of private methods because language has the philosophy: â€“ â€œwe are all adults consenting hereâ€. Meaning that there should be no arbitrary restrictions when accessing parts of a class. Therefore, there are no ways to impose a method or attribute to be strictly private [6]. However, to be able to have the encapsulation feature, Python has a concept of hidden variables in which python does not restrict access but hinders access to certain variables by mutilating them. For Python we have two terms to classify the private variables-weakly private and heavily private methods and attributes. The selection between these two methods of access difficulty is determined by its name.

Weakly private methods and attributes (_single_leading_underscore)

Variables and methods that are weakly private are methods that have an underscore (_) at startup. This type of variable has the name of single leading underscore. This approach ONLY signals that these variables are private and other programmers should not use it in external code. However, the weakly private methods are used much more a conversion and do not prevent the code from being accessed from outside the class. The only real effect is that when the modules are imported the variables with an underscore (_) are not imported [6].

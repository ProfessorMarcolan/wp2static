Object-Oriented Programming in Python: How to use inheritance in Python

25 de June de 201825 de June de 2018 JoaoMarcolanPython, theory

In this easy-to-read, 5 minute post, you will understand how to use Python Inheritance as well as all its jargon. Python Inheritance is super important for code reuse. So, come and understand how to use this feature to leverage your projects.

Main mechanism of inheritance

The primary mechanism of inheritance is to allow a class to be derived from a base class, allowing more specific behavior to be implemented in the subclass. Inheritance is also an important feature for reusing algorithms and avoiding redundant codes that may make it difficult to maintain the code base.

Inheritance Jargon

But before we dive deeper, we will discuss, revise, a bit about nomenclatures and jargon. When a class “B” inherits aspects of class “A”:

  • class “B” is a subclass of class “A”; .
  • class “A” is a super class of class “B”;
  • class “B” is a daughter class of class “A”;
  • class “A” is a parent class of class “B”
  • class “A” is the base class
  • class “B” is the derived class
  • the class “B” “… is a …” class “A”;

Fig. 1: Python Heritage Resource Jargon

 

How to use inheritance in Python

To exemplify the inheritance mechanism, we created a Car class, code 1, which has two methods: the initializer (__init__) and the type of motor that has the car (getMotor).Next, we create a Taxi class that will only access, in two different ways, the methods implemented in its parent class, Car . Finally, we will instantiate an object of class Taxi .The first access method is using class name then the dot operator (.) And finally the method name. The second access method is normal access, self object, dot operator (.) And finally the method name.

      # Código 1 - Using the inheritance feature
      class Car():
          def __init__(self):
              print("Created a car")

          def getMotor(self):
              print("I have a combustion engine")

      class Taxi(Car):
          def __init__(self):
              Carro.__init__(self)
              # Criou um carro
              self.getMotor()

      ca = Taxi()
      # Created a car
      # I have a combustion engine

The result, at the command prompt, was first the phrase “Created a car” and then the phrase “I have a combustion car”. The two access methods invoked the methods implemented in the Car class. The first method explicitly accesses the parent class method and the second method can demonstrate that the methods implemented in the parent class are now also part of the child class. When a class inherits another class, it automatically takes all the methods and attributes of the original class [4]. Relation “… is a ..”

In the post about composition we explain, in general terms, the relationship “… has a …”, “… is part of …” For example, a class describing a car will have objects related to the engine , tires, electrical parts, etc. However, there is another important type of relationship: “… it’s a …” For example, a chocolate cake “… is a …” cake, an apple is When this type of relationship is detected it is necessary to use the Inheritance feature.

Super method, for method access of parent class

In code 1, when we created the Taxi class, we discussed 2 methods of accessing the parent class: the method that writes the class name (e.g., Car.getMotor) and the normal method. However, method 1 is not widely used because it does not decouple the class derived from the parent class. Generally the method used to access the parent class is through the super () method. We will in code 2 implement a CarEletric class that derives the Car class and implements a method using this form of access.

        #código 2
        class CarEletric(Car):
            def __init__(self):
                super().__init__()
                self.getMotor()

        ca =  CarEletric()
        # Created a car
        # I have a combustion engine

When running code 2 the result is similar to the result of code 1. The difference now that the super method decouples the two classes. However, the electric car does not have a combustion engine, so it is necessary to override parent class methods.

Overriding parent class methods

The getMotor () method returns as expected, but the electric car has a different engine. It is possible to override any method of the parent class that does not suit the child class (code 4). To do this you need to define the method in the derived class with the same name as the undesirable method. Python will disregard the method of the parent class and will only pay attention to the method defined in the child class.

        #code 4 Overriding parent class methods
        class CarroEletric(Carro):
            def __init__(self):
                super().__init__()
                self.getMotor()

            def getMotor(self):
                print("I have an electric motor")

        ca = CarroEletric()
        # Created a car
        # I have an electric motor

Now, the result was expected that the electric car has an electric motor.

How inheritance organizes the code

In conclusion, inheritance allows us to reuse codes even though some methods need to be implemented slightly differently in the child class. The Car class can be reused and expanded by the Taxi class and can be reused and modified by the CarroEletric class. Generally, some classes take a great amount of time for testing and development.

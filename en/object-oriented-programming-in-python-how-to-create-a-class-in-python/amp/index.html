Python Object Oriented Paradigm

In the object orientation paradigm you write programs from the creation of classes. We can define the classes as a representation, a reproduction or a model. Classes represent situations / things. When you write a class, you define a general behavior that a whole category of objects can have. When you construct an object using the class you automatically equip this object with all the general behaviors. Classes have data and methods. For each object you can set your unique traits [4]. A didactic analogy would be to say that the classes are like the recipes of a cake, the objects are the cake itself, the attributes are the ingredients and the methods are the actions for the preparation of that cake.

From the definition of the class name everything inside the indentation (four spaces) after the prototyping belongs to the class. And the first line of code without the indentation is out of class. Methods builders and destructors in Python

In other languages that have the object orientation paradigm (eg c ++) objects have two special methods: the method that is invoked by the system when creating an object and the method the system calls when deleting an object. This is done because different objects require different steps to be built and destroyed. In a didactic analogy is to think how to think of the difference between the steps required to build a house in relation to a car. And they also have very different steps to be destroyed.

The method that is called to create the object is the constructor and the method that is invoked when it is necessary to delete it is the destructor. In other languages this is a super important topic because programs may have memory leak if these methods are implemented improperly. Python takes a piece of this work / responsibility from the programmer through the idea of the initializer method (__init__).

The __init__ initializer method

Python allows you to instruct the new creation of an object with an initializer method called __init__. The __init__ method is not the constructor. The constructor method calls the ___init__ method if one is implemented. In Python the object is already built when it calls the __init __ () initializer method. The initializer only sets the already created object before that object returns to whoever requested that new object. That is why it is possible to build objects in Python without having to implement this method in your code. In code 3 we will modify the class â€œFirstClassâ€ to appear with the method __init__. And again we will instantiate an object using this class.

This is because when we instantiate a new object Python invokes the constructor of the object that in turn calls our initializer method. And in our initializer method has a write function on the screen (print) that has as input a string argument. Looking closely at our __init __ () method also has an input argument the self.
